## Overview

PixelCart is a production-ready e-commerce web application that allows users to browse, filter, and purchase premium electronics вЂ” mobiles and laptops from brands like Apple, Samsung, and OnePlus. It features passwordless OTP-based authentication, a full shopping cart, Razorpay payment gateway integration, product reviews and ratings, and a complete admin dashboard for product and order management.

---

## Features

### Authentication
- Passwordless OTP-based authentication вЂ” no passwords stored
- OTP sent to email via Nodemailer (Gmail)
- OTP expires in 10 minutes
- JWT token stored in HTTP-only cookie with 7-day session
- Role-based access control вЂ” user and admin roles

### Store
- Product listing with category filters (Mobiles / Laptops)
- Sort by price (low to high, high to low) and top rated
- Individual product detail page with full specifications
- Star rating and review system (one review per user per product)
- Real-time average rating calculation on every review

### Cart and Checkout
- Add to cart with quantity controls
- Persistent cart state via Redux
- Order summary with itemwise breakdown
- Razorpay payment gateway integration (test mode)
- Order confirmation page after successful payment
- Order history page for each user

### Admin Dashboard
- Protected admin-only routes
- Add, edit, delete products with full spec support
- View all orders with user details and payment status
- Stats overview вЂ” total products, orders, revenue, users

## API Endpoints

### Auth Routes вЂ” `/api/auth`
| Method | Endpoint | Access | Description |
|---|---|---|---|
| POST | `/register` | Public | Register with username and email, sends OTP |
| POST | `/send-otp` | Public | Send OTP for login |
| POST | `/verify-otp` | Public | Verify OTP, issues JWT cookie |
| POST | `/logout` | Public | Clears JWT cookie |

### Product Routes вЂ” `/api/products`
| Method | Endpoint | Access | Description |
|---|---|---|---|
| GET | `/` | Public | Get all products with optional filters |
| GET | `/:id` | Public | Get single product |
| POST | `/` | Admin | Create new product |
| PATCH | `/:id` | Admin | Update product |
| DELETE | `/:id` | Admin | Delete product |

### Order Routes вЂ” `/api/orders`
| Method | Endpoint | Access | Description |
|---|---|---|---|
| POST | `/` | User | Create Razorpay order |
| POST | `/verify-payment` | User | Verify payment and confirm order |
| GET | `/my-orders` | User | Get logged in user's orders |
| GET | `/all` | Admin | Get all orders |

### Review Routes вЂ” `/api/products/:id/reviews`
| Method | Endpoint | Access | Description |
|---|---|---|---|
| GET | `/` | Public | Get all reviews for a product |
| POST | `/` | User | Submit a review |

## Getting Started

### Prerequisites
- Node.js 18+
- MongoDB Atlas account
- Gmail account with App Password enabled
- Razorpay account (test mode)

### Clone the repository

```bash
git clone https://github.com/akshitmalia/PixelCart-Ecommerce.git
cd PixelCart-Ecommerce
```

### Setup backend

```bash
cd server
npm install
```

Create `server/.env` вЂ” see [Environment Variables](#environment-variables) section.

```bash
npm run dev
```

Backend runs on `http://localhost:5000`

### Setup frontend

```bash
cd client
npm install
```

Create `client/.env.local`:
```
NEXT_PUBLIC_RAZORPAY_KEY_ID=your_razorpay_test_key
```

```bash
npm run dev
```

Frontend runs on `http://localhost:3000`

## Deployment

| Service | Platform | URL |
|---|---|---|
| Frontend | Vercel | [pixel-cart-ecommerce.vercel.app](https://pixel-cart-ecommerce.vercel.app) |
| Backend | Render | [pixelcart-backend-6069.onrender.com](https://pixelcart-backend-6069.onrender.com) |
| Database | MongoDB Atlas | Cloud hosted |

### Deploy backend on Render
1. Connect GitHub repo
2. Set Root Directory to `server`
3. Build command: `npm install`
4. Start command: `node server.js`
5. Add all environment variables from `server/.env`

### Deploy frontend on Vercel
1. Connect GitHub repo
2. Set Root Directory to `client`
3. Framework: Next.js (auto detected)
4. Add `NEXT_PUBLIC_RAZORPAY_KEY_ID` environment variable
